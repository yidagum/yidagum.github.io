这一年来做爬虫和一些东西的闲谈.

发表于 2020-09-25 | | 本文总阅读量 次

可能和大多数人一样，我的第一个爬虫代码是这样的。

就这样，人生中可能非常有成就感的事就出现了，一个页面的文字信息通过这短短几行的代码实现了豆瓣电影top250的电影名的自动化爬取，并且保存到了本地，也不知道为啥要添加header头部信息，只知道不添加就不能爬取到目标网页。也不知道为什么要用beautifulsoup进行解析html进行提取。
于是乎，你可能依赖着这种模板式的操作一直尝试着多个网页，百度贴吧，各种图片网站，各种小说网站等等。

可能更进一步会发现，一般的网页信息会出现多页保存，这时候你可能会尝试着进行构造网页参数进行多页爬取，通过for循环实现遍历网页爬取。

在这个时候，一种“会”爬虫的概念从心中升起，渐渐通过进行多个不同网页爬取，你会发现有些网页信息会出现登录才能获取。

从这个时候，就开始不断百度如何实现模拟登录，例如selenium自动化框架实现浏览器模拟操作。可是这是一种矛盾，正如requests作者的官方简介 “HTTP for Humans”所说，操纵selenium进行浏览器模拟，仿佛与按键精灵脚本一般完全变成了一个自动执行的脚本而不是那种可以后台执行的程序。从这个时候你就开始重新进行系统的抓包，通过不断测试找寻维持登录的接口，而且开始了解https的get与post的请求的含义，同时，对会话维持会有一些感悟，并且能够深深感受到requests的强大。于是，可能通过如下代码实现模拟登录

到这个时候，就会出现一种对效率追求的极致，逐渐的不断尝试着用各种多进程线程进行重构自己之前写的爬虫，这个时候，通过自己改写容易出现死锁而且针对不同网页需要分别进行不同的控制，这时候一种可谓神器的’scrapy‘的出现了.

优雅的结构设计，强大的扩展，以及异步强大的效率，这个时候，当爬取效率到达极限，会发现自己的get请求出现大量的403，届时访问目标网站，出现了ip被ban的问题，这时候就又开始找一些通过构造ip代理池进行随机切换代理ip进行访问爬取，还有对分布式逐渐有了点尝试，通过虚拟机安装redis，通过将url作为key进行通信访问爬取。

以上大概是我接触怕爬虫快两年的所有操作，我在19年下学期软件工程课上同老师交流，当我在不断吹嘘自己“会”爬虫的时候，老师给了来自灵魂的质疑“单单爬取数据有什么用？”

逐渐，我也开始思考，爬取的数据到底有什么用，也不知道怎么回事，我在19年上学期意外得了水痘被迫回家隔离治疗，看了一本《数学之美》，作者展示了数学和信息论在自然文本处理上的具体应用，于是乎，19年暑假买了本李航《统计学习方法》看了一暑假仿佛有点进入状态。

在老师灵魂质疑之后，我大概花了好几个晚上，做了一个文本情感分类的扩展，但当时可能主要还是为了应付一学期九门考试中的一门…直接调用snownlp，做了个print输出和一些可视化图表绘制.半年之后，因为snownlp的文本依赖基于淘宝评价，对于我当时爬取的b站弹幕文本不太适用，准确率不是太高。半年后我通过不断查资料各种学习，自己标注了5000弹幕文本，还自己尝试了一下snownlp的朴素贝叶斯分类的实现，结果还是不错，达到93%。

以上，大概是我这一年多学习的东西，有的时候为了一次次应付考试，大部分时间都投入到复习考试，结果还可以，成绩不算太烂，但有的时候真的挺不快乐，但是我综合来看，上学的无忧无虑可能更是我想追求的。
